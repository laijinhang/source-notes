# 一、时间堆
### 1、数据结构
```go
// runtime/runtime2.go
type p struct {
    timersLock mutex
    timers []*timer
    timer0When uint64
    numTimers uint32
}
```
* P.timersLock：定时器锁，保护P.times字段
* P.timers：时间堆，存放了绑定在P上的定时器列表，必须持有timersLock才能访问
* P.timer0When：最先触发的定时器的when绑定在P上的timer0When字段上，如果这个字段的值是0，表示这个定时器堆是空的，这个字段是用原子操作进行修改的
* P.numTimers：P的堆中定时器数量，这个字段是通过原子操作进行修改的
